## 1. 제품 개요

### 비전
옵시디언(Obsidian) 볼트를 구글드라이브와 싱크하여, 모바일에서도 옵시디언 마크다운을 자연스럽게 읽을 수 있는 크로스플랫폼 리더 앱.

### 문제 정의
- 옵시디언 모바일 앱은 구글드라이브 싱크를 공식 지원하지 않음 (iCloud, Dropbox만)
- 구글드라이브의 .md 파일은 뷰어가 없어 날것으로 보임
- 위키링크 `[[노트명]]`, 백링크, 태그 등 옵시디언 특유의 기능을 지원하는 모바일 리더가 없음
- 기존 마크다운 에디터는 옵시디언 볼트 구조(폴더, 링크)에 최적화되지 않음

### 타겟 사용자
- 옵시디언을 데스크톱에서 사용하며 구글드라이브에 볼트를 저장하는 사용자
- 모바일에서 볼트를 **읽고 탐색**하고 싶은 사용자
- 오프라인 환경(출퇴근, 여행 등)에서도 노트 접근이 필요한 사용자

### 핵심 가치 제안
"구글드라이브의 옵시디언 볼트를, 옵시디언처럼 읽자"

## 2. MoSCoW 분류

### 🔴 MUST (v1 — MVP)

| ID | 기능 | 설명 |
|----|------|------|
| M-01 | 구글드라이브 OAuth2 인증 | Google Sign-In으로 계정 연결. API v3 사용 |
| M-02 | 폴더 선택 및 싱크 | 특정 구글드라이브 폴더(볼트 루트) 선택 → 파일 목록 동기화 |
| M-03 | 마크다운 렌더링 | GFM 마크다운 렌더링 (제목, 리스트, 표, 코드블록, 이미지) |
| M-04 | 위키링크 인식 및 탐색 | `[[노트명]]` 파싱 → 탭으로 열기 (같은 볼트 내 검색) |
| M-05 | 오프라인 캐시 | 싱크된 파일 로컬 저장 → 오프라인에서도 열람 가능 |
| M-06 | 다크/라이트 테마 | 시스템 설정 따르기 + 수동 전환 |
| M-07 | 파일/폴더 트리 탐색 | 옵시디언 볼트의 폴더 구조를 트리로 표시 |

### 🟡 SHOULD (v1.5)

| ID | 기능 | 설명 |
|----|------|------|
| S-01 | 백링크 목록 | 각 노트 하단에 해당 노트를 참조하는 노트 목록 표시 |
| S-02 | 태그 인식 및 검색 | `#tag` 파싱 → 태그별 노트 목록 |
| S-03 | 전체 텍스트 검색 | 볼트 내 노트 내용 검색 |
| S-04 | 증분 싱크 | 변경된 파일만 동기화 (전체 다운로드 방지) |
| S-05 | 이미지 표시 | 마크다운 내 `![[image.png]]` 및 `![](url)` 렌더링 |
| S-06 | 코드 하이라이트 | 언어 지정 코드블록 문법 강조 |

### 🟢 COULD (v2)

| ID | 기능 | 설명 |
|----|------|------|
| C-01 | 기본 편집 | plaintext 수준 마크다운 편집 + 구글드라이브 업로드 |
| C-02 | 그래프 뷰 | 위키링크 기반 노트 간 관계 시각화 |
| C-03 | 즐겨찾기 / 북마크 | 자주 보는 노트 고정 |
| C-04 | 다중 볼트 지원 | 여러 구글드라이브 폴더를 볼트로 등록 |
| C-05 | 공유 기능 | 노트 내용을 다른 앱으로 공유 |

### 🔵 WON'T (명시적 배제)

| ID | 항목 | 이유 |
|----|------|------|
| W-01 | 실시간 공동 편집 | 구글드라이브 API로 구현 복잡도 과다 |
| W-02 | 플러그인 시스템 | 옵시디언 호환성이 아닌 독립 앱으로 접근 |
| W-03 | PDF/워드 파일 뷰어 | 마크다운 리더에 집중 |
| W-04 | 유료 결제 / 구독 | 초기 모델은 무료 |

## 3. 사용자 시나리오

### 시나리오 1: 첫 실행 (M-01, M-02, M-07)
1. 앱 실행 → 구글 계정 로그인 (OAuth2)
2. 구글드라이브 내 옵시디언 볼트 폴더 선택
3. 폴더 구조 스캔 → 파일 목록 동기화
4. 폴더 트리 표시

### 시나리오 2: 노트 읽기 (M-03, M-04)
1. 폴더 트리에서 노트 탭
2. 마크다운 렌더링된 내용 표시
3. `[[다른 노트]]` 링크 탭 → 새 탭으로 열기
4. 뒤로가기로 이전 노트 복귀

### 시나리오 3: 오프라인 사용 (M-05)
1. Wi-Fi 환경에서 볼트 싱크 완료
2. 지하철(오프라인)에서 앱 실행
3. 캐시된 노트 정상 열람
4. 링크 탭 → 캐시된 노트만 열기, 미캐시 노트는 안내

### 시나리오 4: 테마 전환 (M-06)
1. 설정에서 다크/라이트/시스템 자동 선택
2. 즉시 적용

## 4. 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| 프레임워크 | Flutter 3.x | iOS + Android 크로스플랫폼 |
| 언어 | Dart 3.x | null safety |
| 인증 | Google Sign-In (OAuth2) | google_sign_in 패키지 |
| 구글드라이브 | Google Drive API v3 (REST) | googleapis 패키지 |
| 상태관리 | Riverpod 2.x | |
| 로컬 DB | Isar (또는 drift/SQLite) | 파일 메타데이터 + 인덱스 |
| 마크다운 | flutter_markdown + 커스텀 확장 | 위키링크, 태그 파싱 |
| 파일 캐시 | path_provider + 로컬 파일시스템 | |
| 테스트 | flutter_test + mockito | TDD |
| CI | GitHub Actions | Lint + 테스트 자동화 |

### Google Drive API 인증 플로우

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│  앱 실행  │────▶│ Google Sign-In│────▶│  Access Token │
│          │     │  (OAuth2)     │     │  + Refresh    │
└──────────┘     └──────────────┘     └──────┬───────┘
                                              │
                                              ▼
                                      ┌──────────────┐
                                      │ Drive API v3  │
                                      │ files.list    │
                                      │ files.get     │
                                      └──────┬───────┘
                                              │
                                              ▼
                                      ┌──────────────┐
                                      │ 로컬 DB 저장  │
                                      │ (Isar/SQLite) │
                                      └──────────────┘
```

- google_sign_in으로 사용자 인증 → access_token + refresh_token 획득
- googleapis 패키지로 Drive API v3 호출
- 토큰은 flutter_secure_storage에 암호화 저장
- refresh_token으로 자동 갱신

## 6. 성공 기준

| 지표 | 목표 |
|------|------|
| 위키링크 탐색 응답 | 탭 후 200ms 이내 렌더링 |
| 오프라인 접근 | 싱크된 모든 노트 100% 열람 |
| 첫 싱크 (500노트 기준) | 30초 이내 완료 |
| 증분 싱크 | 변경분만, 5초 이내 |
| 앱 크기 | 20MB 이하 (이미지 제외) |

## 7. 리스크 및 제약

| 리스크 | 대응 |
|--------|------|
| 구글드라이브 API 할당량 | 사용자당 10GB/일, 증분 싱크로 최소화 |
| iOS Google Sign-In 설정 | Firebase 프로젝트 필요, Reversed Client ID 설정 |
| 위키링크 파싱 복잡도 | `[[파일명]]`, `[[파일명\|별칭]]`, `[[폴더/파일명]]` 케이스 정의 필요 |
| 옵시디언 특수 문법 | callout(`> [!note]`), dataview 등은 v2에서 검토 |
| Flutter 마크다운 제한 | flutter_markdown 한계 시 webview + markdown-it 대안 |
